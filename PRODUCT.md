# Product

## Register

product

## Users

People who write posts, replies, messages, and other short text in a browser.
They want to begin in their own words and improve the writing before they
publish it.

## Product Purpose

Plain Writing gives the user one small launcher on each normal web page. The
launcher opens a private writing panel above the current page. The user pastes
text, asks for a rewrite, reviews the result, and copies it back to any site.

The first version does not inspect text fields, read page content, or replace
text automatically. Success means the user can improve text without leaving
the page and without wondering which field the extension is watching.

## Brand Personality

Direct, trustworthy, and modern. The product looks polished and current, but it
stays calm and does not compete with the page behind it.

## Anti-references

The interface should not look like a chat application, an attention-seeking AI
assistant, or a floating button attached to every text field. It uses one brand
gradient on the launcher and the main action only. It should not add glass
effects, large decorative headings, or playful motion beyond a single gentle
open animation.

## Design Principles

1. Keep the user's words and meaning.
2. Ask before changing anything outside the extension.
3. Keep provider choice open.
4. Show the result before the user copies it.
5. Make privacy and error states clear.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Support keyboard use, visible focus, reduced motion, screen
reader labels, text zoom, and narrow browser windows.
